## Purpose

Defines the observable streaming behaviour of the Claude engine from the orchestrator's perspective: tokens and reasoning fragments are delivered incrementally during generation, and no content is double-emitted when the final assembled message arrives.

## Requirements

### Requirement: Claude engine delivers text tokens incrementally during generation
The system SHALL stream individual text fragments to the orchestrator as they are generated by the model, rather than delivering a single complete message after the turn ends. Each `text_delta` event from the Anthropic streaming API SHALL be translated to a `{ type: "token" }` engine event and forwarded immediately.

#### Scenario: Text tokens arrive incrementally while model is generating
- **WHEN** the Claude engine is executing and the model produces a text response
- **THEN** multiple `{ type: "token" }` engine events are emitted during generation, each containing a fragment of the final text — not a single event containing the full response

#### Scenario: Final assembled text is not double-emitted
- **WHEN** the Claude engine has already emitted incremental `token` events for a turn
- **THEN** the final assembled `assistant` message does NOT produce an additional `token` event for the same text content

---

### Requirement: Claude engine delivers reasoning tokens incrementally during generation
The system SHALL stream individual reasoning/thinking fragments to the orchestrator as they are generated by the model. Each `thinking_delta` event from the Anthropic streaming API SHALL be translated to a `{ type: "reasoning" }` engine event and forwarded immediately.

#### Scenario: Reasoning tokens arrive incrementally while model is thinking
- **WHEN** the Claude engine is executing with a model that produces extended thinking
- **THEN** multiple `{ type: "reasoning" }` engine events are emitted during the thinking phase, each containing a fragment — not a single event containing the full reasoning block

#### Scenario: Final assembled reasoning block is not double-emitted
- **WHEN** the Claude engine has already emitted incremental `reasoning` events for a turn
- **THEN** the final assembled `assistant` message does NOT produce an additional `reasoning` event for the same thinking content

---

### Requirement: Claude engine continues to process tool_use blocks from assembled assistant message
The system SHALL continue to extract `tool_use` blocks from the fully-assembled `assistant` message and emit `tool_start` engine events. Tool calls SHALL NOT be sourced from `input_json_delta` streaming events.

#### Scenario: Tool call produces a tool_start event from the assistant message
- **WHEN** the model's turn includes a tool call
- **THEN** a `{ type: "tool_start" }` engine event is emitted with the correct `callId`, `name`, and `arguments` from the assembled `assistant` message
